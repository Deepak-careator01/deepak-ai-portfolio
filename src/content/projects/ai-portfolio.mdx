## Problem Statement

Traditional portfolios are static brochures. They look polished, but visitors cannot ask questions, explore architecture decisions interactively, or navigate to the most relevant sections based on intent.

I needed a portfolio that demonstrates modern full-stack engineering while also serving as a foundation for AI-native experiences — chat, retrieval, and eventually agentic copilots with tools and memory.

## Solution Approach

The solution is a content-driven Next.js application where portfolio data lives in structured TypeScript and MDX files. The UI, SEO metadata, RAG pipeline, and future agent tools all consume the same source of truth.

The roadmap progresses in phases:

1. Premium portfolio UI
2. Streaming AI chat
3. RAG over portfolio content
4. LangGraph agent with tools (e.g. `getProject`, `navigateToSection`)

## System Architecture

The application follows a layered architecture:

- **Content layer** — `profile`, `skills`, `experience`, and MDX project case studies
- **Presentation layer** — React Server Components for sections and case study pages
- **AI layer (planned)** — API routes, embeddings store, and LangGraph workflows in `server/`

Content is designed for chunking: each project case study includes explicit sections (problem, architecture, decisions, impact) that map cleanly to retrieval units.

## Technical Stack

- **Frontend:** Next.js 15 App Router, React 19, Tailwind CSS v4, shadcn/ui
- **Runtime:** Bun
- **AI (planned):** Vercel AI SDK, LangGraph.js, OpenAI or Gemini
- **Data (planned):** PostgreSQL, Drizzle ORM, pgvector

## Key Features

- Dark-first premium UI with responsive sections
- MDX-backed project case studies with dynamic routes
- Centralized metadata generation for SEO
- Strict TypeScript content models shared across UI and AI pipelines
- Copilot placeholder architecture from day one

## Challenges

- Keeping MDX content, TypeScript types, and UI components in sync without duplication
- Designing schemas that work for both human-readable case studies and machine retrieval
- Maintaining performance while preparing for streaming chat and agent tooling

## Engineering Decisions

- **RSC-first:** Minimize client JavaScript; isolate interactivity to theme toggle and future copilot UI
- **MDX in `content/projects/`:** Rich case studies with version-controlled frontmatter for filtering and agent tools
- **Separate loaders in `lib/projects.ts`:** File I/O and MDX compilation stay out of UI components
- **Phased AI integration:** Ship a strong portfolio foundation before adding RAG and agents

## Performance or Business Impact

- Static generation for project pages via `generateStaticParams`
- Server-side MDX compilation avoids shipping MDX parsers to the client
- Positions the portfolio as a living demo of AI engineering capability for recruiters and collaborators

## Future Improvements

- Automated ingest script to embed content into pgvector
